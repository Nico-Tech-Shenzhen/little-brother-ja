# あとがき — ブルース・シュナイアー

私はセキュリティ技術者である。人々を安全にすることが私の仕事だ。

私は、セキュリティシステムについて、そしてそれをどう破るかについて考える。そして、それをどうすればより安全にできるかを考える。コンピューターのセキュリティシステム。監視システム。航空機のセキュリティシステム、投票機、RFID（Radio-Frequency Identification）チップ、その他あらゆるものについて。

Coryが本書の最後の数ページに私を招いてくれたのは、セキュリティは楽しいものだということを、私からみなさんに伝えてほしかったからだ。実際、これはとてつもなく楽しい。ネコとネズミの追いかけっこであり、誰が誰を出し抜けるかという駆け引きであり、狩る者と狩られる者の緊張感を楽しむゲームだ。私は、これほど楽しい仕事は他にないと思っている。もし、マーカスが靴に石を入れて歩行認識カメラを出し抜く場面を読んで楽しいと思ったなら、その手口を世界で最初に思いついた人間になれたら、どれほど楽しいか想像してみてほしい。

セキュリティの仕事をするということは、テクノロジーについて多くを知っているということだ。コンピューターやネットワークについて、カメラとその仕組みについて、あるいは爆発物探知の化学について知っていることかもしれない。しかし本当のところ、セキュリティとは一つの心の持ちよう、一つの考え方である。マーカスは、その考え方の見事な実例だ。彼は常に、セキュリティシステムがどこで破綻するかを探している。おそらく彼は、店に入っただけで万引きの方法を考えついてしまうタイプだろう。実際にやるかどうかは別の話だ――セキュリティシステムを破る方法を知っていることと、実際に破ることの間には違いがある――でも、やろうと思えばできる、ということは彼自身わかっているはずだ。

これがセキュリティの専門家の考え方だ。私たちは常にセキュリティシステムと、それをどう回避できるかを見ている。そうせずにはいられないのだ。

この種の考え方は、セキュリティのどちら側に立っていようと重要だ。万引きされない店を作るよう依頼されたなら、万引きの方法を知っておいたほうがいい。個人の歩き方を検知するカメラシステムを設計しているなら、靴に石を入れる人がいることを想定しておいたほうがいい。そうしなければ、ろくなものは設計できない。

だから、日々の暮らしの中でふと立ち止まって、身の回りのセキュリティシステムに目を向けてみてほしい。買い物をする店の防犯カメラを見てみる（それは犯罪を防いでいるのか、それとも隣の店に追いやっているだけなのか？）。レストランの仕組みを観察する（食べたあとに支払うのなら、なぜもっと多くの人が黙って店を出て行かないのか？）。空港のセキュリティに注意を払う（どうすれば武器を飛行機に持ち込めるだろうか？）。銀行の窓口係の様子を見る（銀行のセキュリティは、客の盗みを防ぐのと同じくらい、窓口係自身の盗みを防ぐために設計されている）。アリの巣をじっと見つめる（昆虫の世界はセキュリティだらけだ）。憲法を読んで、それが政府に対する国民のセキュリティをどのように保障しているか、あらゆる面で確かめてみる。信号機やドアの鍵、テレビや映画に出てくるあらゆるセキュリティシステムを見てみる。それがどう機能するのか、どんな脅威から守っていてどんな脅威からは守っていないのか、どう破綻するのか、どう悪用されうるのかを考えてみてほしい。

これを十分な時間をかけて続けていれば、いつの間にか世界の見方が変わっていることに気づくはずだ。世の中の多くのセキュリティシステムが、実際には謳っている通りの働きをしていないこと、そして国家安全保障の多くが金の無駄遣いであることに気づき始めるだろう。プライバシーはセキュリティと対立するものではなく、その本質的な一部であることが理解できるようになる。他の人々が心配していることを心配しなくなり、代わりに、他の人々がそもそも考えもしないことを心配するようになるだろう。

時には、これまで誰も考えたことのないセキュリティ上の何かに気づくこともあるだろう。そして、セキュリティシステムを破る新しい方法を発見するかもしれない。

フィッシングという手口が発明されたのは、ほんの数年前のことだ。

かなり名の知れたセキュリティシステムでさえ、いかに簡単に破れてしまうかに、私はしばしば驚かされる。理由はいろいろあるが、一番大きいのは、あるものが「安全である」ことを証明するのは不可能だという点だ。できるのは、それを破ろうと試みることだけだ――もし破れなければ、少なくとも*自分自身*を締め出すには十分安全だとわかる。でも、自分より賢い誰かが相手だったらどうだろう？ 誰でも、自分自身では破れないほど強固なセキュリティシステムを設計することはできる。

これは自明なことではないので、少し考えてみてほしい。誰も、自分自身のセキュリティ設計を分析する資格はない。なぜなら、設計者と分析者が同一人物であれば、同じ限界を抱えたままになるからだ。設計者が思いつかなかった脅威に対しても安全でなければならない以上、セキュリティの分析は別の誰かがやらなければならない。

つまり、私たち全員が、他の誰かが設計したセキュリティを分析しなければならない、ということだ。そして驚くほど頻繁に、私たちの誰かがそれを破ってしまう。マーカスがやってのけたことは、決して突飛な話ではない。この種のことは、現実にしょっちゅう起きている。ネットで「バンプキー」や「Bicペンでクリプトナイト錠を開ける」などと検索してみてほしい。一見強固に見えるセキュリティが、驚くほど基本的な技術で破られた興味深い話がいくつも見つかるはずだ。

そして、そういうことが起きたら、必ずインターネットのどこかで公開してほしい。秘密にすることとセキュリティは、そう見えるかもしれないが、同じものではない。悪いセキュリティだけが秘密に依存する。良いセキュリティは、詳細をすべて公開しても機能する。

脆弱性を公開することは、セキュリティ設計者により良いセキュリティを設計するよう強い、私たち全員をより賢いセキュリティの消費者にしてくれる。もしクリプトナイトの自転車錠を買って、それがBicペン一本で破られてしまうなら、あなたはお金に見合ったセキュリティを手にしていないことになる。それと同じように、もし賢い子どもたちの一団がDHS（国土安全保障省）の対テロ技術を出し抜けてしまうなら、それは本物のテロリストに対してもろくに役に立たないということだ。

プライバシーとセキュリティを引き換えにすること自体すでに愚かなことだが、その取引で実際のセキュリティすら手に入らないとしたら、それはもっと愚かなことだ。

だから、この本を閉じて、外に出よう。世界はセキュリティシステムであふれている。その一つをハックしてみよう。

Bruce Schneier
http://www.schneier.com
